### Постановка задачи
>Разработать систему управления списком покупок с функциями анализа бюджета и
>стоимости товаров. Обеспечить возможность работы с данными о товарах, их ценах и
>доступных средствах, включая проверку соответствия стоимости покупок бюджету

#### Функционал:
>1. Создание списка покупок
>Пользователь может создать список товаров с указанием их названий, цен, статусов «куплено/не куплено» и необходимого количества.
>2. Указание бюджета
>При создании списка можно задать начальную сумму денег, доступных для покупок. Если бюджет не указан, он равен нулю.
>3. Добавление денег
>Возможность увеличить доступный бюджет на указанную сумму.
>4. Проверка возможности покупки товара
>Пользователь может проверить, достаточно ли у него денег для покупки определённого товара. Поиск товара может осуществляться по его номеру или названию.
>5. Покупка товара
>Пользователь может приобрести товар из списка, если у него достаточно средств.
>6. Общая стоимость списка
>Вычисление суммы всех товаров в списке, независимо от их статуса (куплено/не
>куплено).
>7. Общая стоимость оставшихся товаров
>Подсчёт стоимости всех товаров, которые ещё не куплены.
>8. Возможность покупки всего списка
>Проверка, можно ли купить все товары из списка при текущем бюджете.
>9. Подсчёт недостающей суммы
>Если средств недостаточно для покупки списка товаров, отображается сумма, которая требуется для полной покупки. При наличии лишних средств результат не
>должен уходить в отрицательные значения.
>10. Функция «Купить всё»
>Попытка автоматически приобрести все товары из списка, начиная с самого дешёвого или дорогого, в зависимости от настроек.
>11. Сортировка товаров по цене
>Возможность упорядочить товары в списке по их стоимости, начиная с самого
>дешёвого или самого дорогого.
>12. Покупка определённого количества товаров
>Пользователь может определить, сколько товаров из списка он может купить,
>начиная с самых дешёвых или самых дорогих, в зависимости от настроек.
>13. Функция нахождения самого дешёвого или дорогого товара
>Вывод информации о самом дешёвом или дорогом не купленном товаре для упрощения взаимодействия.

Для реализации постановленной задачи я создала три класса: Бюджет, Товар и Список. Далее подробно описываю, как они работают.

### Класс "Бюджет"

Данный класс хранит в себе информацию о текущем бюджете в переменной `sum`. Он реализует следующие функции: указание бюджета, вывод бюджета типа `double`, изменение/увеличение бюджета.

Пользователь может задать бюджет, изменять его при необходимости, выводить текущий бюджет на экран. При покупке бюджет, очевидно, уменьшается.

### Класс "Товар"

Данный класс хранит в себе информацию о конкретном товаре, который пользователь может добавить в список. У товара 4 характеристики: название (`name`), цена (`cost`), статус куплено/не куплено (`status`) и количество (`amount`).

Класс позволяет создать новый товар, и возвращает стоимость товара (для работы остальной программы неоходимой оказалась только эта функция). После создания товара он добавляется в список покупок.

### Класс "Список"

Это основной класс для создания и работы со списком покупок. Он задается массивом элементов класса "Товар" (я поставила ограничение - 100 объектов) и переменной `k`, необходимой для учета количества элементов в списке.

Класс реализует следующие функции: создание нового пустого списка, добавление туда товара, сортировку списка по цене товара, подсчет стоимости всего списка и стоимости некупленных товаров, возможность совершить покупку, попытку купить определенное количество товаров и попытку купить все товары, вывод на экран самого дешевого и самого дорогого товара.

Если не хватает бюджета на ту или иную покупку, программа не позволяет этого сделать (бюджет не уходит в минус).

### Тест

В тестовой программе я проверяю работу всех функций. Реализован довольно интерактивный вывод, чтобы точно можно было определить работу той или иной функции. Для этого создается ряд новых списков.
